SAFE Project Using F#

By Cromwell Cutaran, Monil Gandhi, Kristie Kwak, Ivana Yu

Overview of F#

  • Not a pure functional programming language
  • Like Scala, can treat functions as values, have curried functions, etc
  • mutable keyword to change a mutable variable's value

Pros of F#

  • Since it's still a functional programming language, supports immutable data structures
  • Concise syntax
  • Pipe operators make code more readable because you don't have to store intermediate results
  • There is a Record data types (similar to a Java class but records are comparable) 
           type MyRecord = {
          X: int;
          Y: int;
          Z: int
       }
       let myRecord1 = { X = 1; Y = 2; Z = 3; }
  • Pattern matching feature 
     
          match expression with
             | pattern [ when condition ] -> result-expression
             ...
    "Each pattern acts as a rule for transforming input in some way. In the match expression, each pattern is examined in turn to see if the input data is compatible with the pattern"

More Pros

  • Easy to create new types
  • Good for asynchronous workflows 
              async { expression }
    where expression is executed asynchronously when you call a starting function
  • Good IDE support in Visual Studio
  • F# runs on the Common Language Runtime (CLR) so there's access to the .NET Framework
  • SAFE uses Fable, which is an F#-to-Javascript compiler. Lets you share code between client and server

Cons of F#
